# /audit-neutrality - Neutrality Verification

Three-agent debate to verify briefing neutrality.

## Usage

```
/audit-neutrality <briefing-dir>
```

## Process

Three agents review the briefing:

1. **Critic** - Identifies potential bias, advocacy, or imbalance
2. **Defender** - Argues for the objectivity of flagged content
3. **Arbiter** - Makes final determination with remediation steps

## Checks

- Verdict language (confirmed, debunked, proven, false)
- Advocacy framing (must, should, need to, clearly)
- Emotional language (shocking, outrageous)
- Balanced perspective coverage
- Proper attribution

## Output

- List of issues found with severity
- Required remediation steps
- Pass/fail determination

## Gate 6 Criteria

All critical issues resolved, balanced presentation verified.
